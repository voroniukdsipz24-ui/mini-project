# Посібник користувача — Hotel Booking System

## Вимоги

- .NET 8 SDK або Runtime
- Сучасний браузер (Chrome, Firefox, Edge, Safari)
- Windows / macOS / Linux

## Запуск

```bash
git clone <repo-url>
cd HotelBooking
dotnet run --project src/HotelBooking.Web
```

Відкрийте браузер: **http://localhost:5000**

При першому запуску автоматично створюється папка `data/` з демонстраційними даними:
- 9 номерів (Standard, Deluxe, Suite, Penthouse)
- 3 гості

---

## Інтерфейс

### Шапка
- Логотип «Grand Palais» зліва
- Поточна дата і статус API («Online» / «Offline») справа

### Сайдбар (зліва)
- **Головна**: Дашборд / Бронювання / Номери / Гості
- **Аналітика**: Звіти
- **Дії**: Швидкі кнопки «Нове бронювання» і «Новий гість»

### Основна область
Контент змінюється залежно від обраного розділу.

---

## Розділ «Дашборд»

Чотири картки зверху:
- **Завантаженість** — % зайнятих номерів (Occupied + Reserved)
- **Активних броней** — Confirmed + CheckedIn
- **Гостей** — загальна кількість зареєстрованих
- **Загальний дохід** — сума по всіх не-скасованих бронюваннях

Знизу — таблиця останніх 6 бронювань з кнопкою «Всі →» переходу до повного списку.

---

## Розділ «Бронювання»

### Створення бронювання
1. Натисніть «**+ Нове бронювання**» (праворуч зверху або у сайдбарі)
2. У модальному вікні оберіть:
   - **Гостя** з dropdown
   - **Номер** (показуються лише доступні)
   - **Дату заїзду** і **виїзду**
3. Внизу форми показується live-розрахунок ціни: `N ночей × ціна × мультиплікатор сезону = сума`
4. Натисніть «Створити бронювання»

### Управління бронюванням
У таблиці бронювань остання колонка містить кнопки залежно від статусу:

| Статус | Доступні кнопки |
|--------|----------------|
| Pending | Підтвердити, Скасувати |
| Confirmed | Check-in, Скасувати |
| CheckedIn | Check-out |
| CheckedOut | — (фінальний стан) |
| Cancelled | — (фінальний стан) |

При скасуванні з'явиться prompt для введення причини.

### Фільтрація
- **Текстовий пошук** — по гостю або номеру
- **Dropdown статусу** — фільтр за статусом бронювання

---

## Розділ «Номери»

Сітка карток з усіма номерами. Кожна картка показує:
- Номер кімнати (велике число)
- Тип і поверх
- Статус (badge кольоровий)
- Ціна за ніч
- Місткість (👤 N)
- Опис

Фільтри: тип номера + статус.

---

## Розділ «Гості»

Таблиця всіх гостей з пошуком по імені/email. Колонка «Бронювань» показує кількість бронювань цього гостя.

### Реєстрація нового гостя
1. Натисніть «**+ Новий гість**»
2. Заповніть форму: ім'я, прізвище, email, телефон, паспорт, дата народження
3. Email повинен бути унікальним — система не дозволить зареєструвати двох гостей з однаковою адресою

---

## Розділ «Звіти»

Три блоки аналітики:

### 1. Дохід за типами номерів
Горизонтальні бари: Standard / Deluxe / Suite / Penthouse → сума доходу за весь період.

### 2. ТОП гостей за витратами
Топ-5 гостей з найбільшими витратами (горизонтальні бари золотого кольору).

### 3. Зайнятість за місяцями
Стовпчастий графік: 12 місяців × кількість зайнятих ліжко-ночей.

---

## Розрахунок ціни (PricingEngine)

Формула: **Базова ціна × Кількість ночей × Мультиплікатор сезону**

> Базова ціна за ніч уже відображає клас номера (Standard / Deluxe / Suite / Penthouse) — кожен номер має свою власну ціну. Додатковий type-мультиплікатор не застосовується.

| Сезон | Місяці | Множник |
|-------|--------|---------|
| Пік | Червень, Липень, Серпень, Грудень | × 1.25 |
| Шолдер | Березень, Квітень, Вересень, Жовтень | × 1.10 |
| Off-peak | Січень, Лютий, Травень, Листопад | × 1.0 |

---

## Помилки та їх значення

| Toast (червоний) | Причина |
|---|---|
| `Room N is not available...` | Номер зайнятий на вказані дати |
| `Guest #N not found` | Гостя з таким ID не існує |
| `Cannot cancel an active...` | Гість вже заселений (CheckedIn) |
| `Guest with email '...' already exists` | Email вже зареєстрований |
| `Booking must be confirmed before check-in` | Спроба check-in для Pending бронювання |

## Зберігання даних

Файли у папці `data/` поруч з виконуваним файлом:
- `bookings.json` — бронювання
- `rooms.json` — номери
- `guests.json` — гості

Запис атомарний (через тимчасовий файл + rename) — навіть при аварійному завершенні файли не пошкодяться.

Для скидання до demo-даних — видаліть папку `data/` і перезапустіть програму.
